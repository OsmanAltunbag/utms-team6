# SPEC-014 — Approve English Proficiency
## Maps to: UC-05-01

## Goal
YDYO officer reviews English proficiency certificate/exam result in-browser, then approves or rejects. Approved → application routes to DEPT_EVAL. Rejected → application eliminated. Full audit logging (SRS SR1).

## Depends On
SPEC-004

---

## Tasks

### Task 1 — EnglishProficiencyService
File: `backend/app/services/english_service.py`

```python
class EnglishProficiencyService:
    async def approve(
        self, application_id: UUID, reviewer_id: UUID,
        exam_type: str, exam_score: float
    ) -> EnglishProficiencyReview
    # status: ENGLISH_REVIEW → DEPT_EVAL
    # AuditLog action="ENGLISH_APPROVED" with reviewer_id + timestamp (SRS SR1)
    # Notify applicant within ≤ 30 seconds (SRS perf)
    # Action processed + saved ≤ 1 second (SRS perf)

    async def reject(
        self, application_id: UUID, reviewer_id: UUID,
        rejection_reason: str, notes: str
    ) -> EnglishProficiencyReview
    # status: ENGLISH_REVIEW → REJECTED
    # AuditLog action="ENGLISH_REJECTED"
    # Notify applicant
```

Predefined rejection reasons (SRS AC-01):
`"EXPIRED_EXAM"`, `"INSUFFICIENT_SCORE"`, `"UNVERIFIABLE_DOCUMENT"`, `"OTHER"`

```
  Body: { rejection_reason, notes }
```

Document preview: reuse `DocumentService.generate_preview_url()`. In-browser PDF viewer mandatory (SRS SR2).

# SPEC-015 — Announce English Proficiency Exam Results
## Maps to: UC-05-02

## Goal
YDYO officer publishes bulk English proficiency exam results. Pass → routes to Dean/Faculty Evaluation. Fail → eliminated. Publication is irreversible. Audit log stores timestamp + officer ID permanently (SRS SR1, SR2).

## Depends On
SPEC-014, SPEC-020 (Notification Worker)

---

## Tasks

### Task 1 — ExamResultsService
File: `backend/app/services/english_service.py` (extension)

```python
async def publish_exam_results(
    self, results: list[ExamResult], officer_id: UUID
) -> PublicationSummary
# results: [{ application_id, score, passed }]
# For each result (atomic per applicant):
#   passed=True  → approve_english (routes to DEPT_EVAL)
#   passed=False → reject_english (eliminated)
# Enqueue notifications for all
# Process at rate of ≥ 100 applications/second (SRS perf)
# AuditLog action="EXAM_RESULTS_PUBLISHED" with officer_id + timestamp (SRS SR2)
# Irreversible: cannot undo bulk operation without SYSTEM_ADMIN (SRS SR1)
# Returns: { processed, passed_count, failed_count }
```

### Task 2 — YDYO Exam Results Router
File: `backend/app/api/ydyo.py`

```
GET /api/ydyo/exam-results
  Auth: YDYO
  Response: [{ application_id, applicant_name, score, passed }]
  Performance: screen loads ≤ 3 seconds (SRS perf)

POST /api/ydyo/exam-results/publish
  Auth: YDYO
  Body: { results: [{ application_id, score, passed }] }
  Response: 200 { processed, passed_count, failed_count }
  Note: Irreversible (SRS SR1). System confirms via dialog before accepting.
```

# SPEC-016 — Approve Transfer Application (Dean's Final Decision)
## Maps to: UC-06-01

## Goal
Dean reviews Faculty Commission evaluation, intibak table, and academic history, then makes the legally binding final decision: Approve → "Transfer Accepted" → forward to Student Affairs registration OR Reject → permanent closure. Immutable post-decision. Digital record with Dean's ID + timestamp + IP (SRS SR1, SR2, SR4).

## Depends On
SPEC-011 (Ranking approved), SPEC-014 (English satisfied)

---

## Tasks

### Task 1 — DeanOfficeService
File: `backend/app/services/dean_service.py`

```python
class DeanOfficeService:
    async def approve_final(
        self, application_id: UUID, approver_id: UUID, ip_address: str
    ) -> Application
    # status → ANNOUNCED ("Transfer Accepted")
    # Lock ALL evaluation data (immutable, SRS SR2)
    # Forward to ÖİDB registration workflow (SRS POST1)
    # AuditLog action="DEAN_FINAL_APPROVED" with dean_id + timestamp + ip (SRS SR4)
    # Notify applicant with next steps (registration dates) within ≤10 seconds (SRS perf)
    # Locking + archiving ≤ 3 seconds (SRS perf)

    async def reject_final(
        self, application_id: UUID, approver_id: UUID,
        rejection_code: str, note: str, ip_address: str
    ) -> Application
    # status → REJECTED ("Transfer Rejected – Dean's Decision")
    # Status marked FINAL and IRREVERSIBLE (SRS AC-01)
    # AuditLog action="DEAN_FINAL_REJECTED" with dean_id + timestamp + ip (SRS SR4)
    # Notify applicant instantly (SRS AC-01, ≤10 seconds, SRS perf)
```

Standardized rejection codes (SRS AC-01):
`"INSUFFICIENT_ACADEMIC_STANDING"`, `"QUOTA_LIMIT_REACHED"`, `"DISCIPLINARY_RECORD"`, `"UNSUITABLE_PROGRAM_MATCH"`, `"OTHER"`
