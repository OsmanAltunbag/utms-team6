# SPEC-010 — Generate Ranking Automatically
## Maps to: UC-04-03

## Goal
System calculates composite transfer score for all eligible applicants and generates primary (asil) + waitlisted (yedek) ranking lists. Formula: Exam Component (90%) + GPA Component (10%). Deterministic, auditable (SRS SR1, SR2).

### Task 1 — Ranking Score Formula (Pure Function)
File: `backend/app/services/ranking_service.py`

From SRS UC-04-03:
```python
def calculate_transfer_score(
    yks_score: float,
    program_base_score: float,
    gpa_100: float
) -> float:
    """
    Official SRS formula:
      Exam Component = (yks_score / program_base_score) × 100 × 0.90
      GPA Component  = gpa_100 × 0.10
      Transfer Score = Exam Component + GPA Component
    Pure function — deterministic (SRS SR1).
    """
    exam_component = (yks_score / program_base_score) * 100 * 0.90
    gpa_component = gpa_100 * 0.10
    return round(exam_component + gpa_component, 3)
```

### Task 2 — RankingService
File: `backend/app/services/ranking_service.py`

```python
class RankingService:
    async def generate_ranking(
        self, program_id: UUID, period_id: UUID, generated_by: UUID
    ) -> Ranking
    # 1. Fetch all DEPT_EVAL-passed applications for program+period
    # 2. Skip applications with missing yks_score or base_score → flag as "Excluded – Missing Score Data" (SRS AC-01)
    # 3. Calculate transfer_score for each eligible applicant
    # 4. Sort descending (tie-break: submitted_at ASC — deterministic, SRS SR2)
    # 5. Assign positions 1..N
    # 6. Positions 1..quota → is_primary=True (Asil)
    # 7. Remaining → is_primary=False (Yedek)
    # 8. Create Ranking (status=DRAFT), RankingEntry records
    # 9. AuditLog action="RANKING_GENERATED"
    # Performance: complete within 10 seconds for 1000 applicants (SRS perf)
```

# SPEC-011 — Approve System-Generated Ranking
## Maps to: UC-04-04

## Goal
YGK reviews composite scores and approves (or returns for correction) the ranking. Approval is atomic, uses SERIALIZABLE isolation + SELECT FOR UPDATE. Post-approval: ranking is immutable (SRS SR3). Approved ranking forwarded electronically to Dean's Office.

## Depends On
SPEC-010

---

## Tasks

### Task 1 — RankingService (extension)
File: `backend/app/services/ranking_service.py`

```python
async def approve_ranking(
    self, ranking_id: UUID, approver_id: UUID
) -> Ranking
# SERIALIZABLE isolation + SELECT FOR UPDATE (concurrent approval protection)
# 1. Lock ranking row
# 2. Validate status == DRAFT → 422 if not
# 3. Update: status=APPROVED, approved_by, approved_at
# 4. All associated apps → change_status(RANKING) via ApplicationService
# 5. AuditLog action="RANKING_APPROVED" with approver_id + timestamp (SRS SR2)
# 6. Forward electronically to Dean's Office (POST notification/event)
# Concurrency conflict → 409 "Ranking was modified concurrently, please retry"
# Immutability enforced post-approval (SRS SR3)
# Complete within 5 seconds (SRS perf)

async def return_for_correction(
    self, ranking_id: UUID, reviewer_id: UUID, note: str
```
